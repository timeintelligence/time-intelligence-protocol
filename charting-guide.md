# Charting Guide

This guide controls preparation. It does not replace the protocol.

## Required Input

Collect the full input package before analysis:

- Gregorian birth date.
- Exact birth time in 24h format.
- Birth city and country.
- Gender.
- True solar time status.
- Four Pillars.
- Hidden stems.
- Luck Pillar table.
- Starting age.
- Current Luck Pillar.
- Annual Pillar.
- Current decision or question.

Do not begin if any required item is missing and cannot be stated as an explicit assumption.

## Data Control

- Use the provided Luck Pillar table.
- Do not recalculate the Luck Pillar table during execution.
- If two sources conflict, stop and request resolution.
- Record unverified assumptions and the evidence that would confirm or refute them.

## Execution Order

Run the protocol in three phases. Only Step 0-8 are the mandatory reasoning chain.

### Phase 1: Setup

- Read Protocol Stance.
- Complete Data Intake.

### Phase 2: Reasoning Chain

Run Steps 0-8 in order. Do not skip, merge, or reorder.

0. Data Verification.
1. Day Master vs. Month.
2. Roots in Branches.
3. Net Force on Day Master.
4. Strength Verdict.
5. Useful Gods - Preliminary.
6. Structural Pattern and Final Useful God.
7. Luck Pillar and Annual Disturbance.
8. Decision Application.

Skipping, merging, or reordering any Step 0-8 invalidates the output.

### Phase 3: Output and Validation

- Apply Output Format.
- Verify against Rules.
- Review falsification awareness and hard-check requirements.

## Output Control

The final answer must contain:

- Full reasoning chain.
- Tendency and Condition.
- If-Then Switches.
- Execution Timing.
- Year-End Hard Check.

Remove any statement that is not traceable to a specific stem-branch interaction or protocol step.
