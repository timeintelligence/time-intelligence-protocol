# Valid Output Checklist

Use this checklist after generation. A failed hard item invalidates the output.

## Protocol Load

- [ ] `protocol-v2.1.md` was loaded in full.
- [ ] The protocol text was not paraphrased as a substitute for execution.
- [ ] Version 2.1 is visible or explicitly named.

## Data

- [ ] Gregorian birth date is present.
- [ ] Exact birth time is present.
- [ ] Birth city and country are present.
- [ ] Gender is present.
- [ ] True solar time status is stated.
- [ ] Four Pillars are present.
- [ ] Hidden stems are present.
- [ ] Luck Pillar table is present.
- [ ] Starting age is present.
- [ ] Current Luck Pillar is present.
- [ ] Annual Pillar is present.
- [ ] Conflicts or assumptions are declared.

## Chain Integrity

- [ ] Step 0 is present.
- [ ] Step 1 is present.
- [ ] Step 2 is present.
- [ ] Step 3 is present.
- [ ] Step 4 is present.
- [ ] Step 5 is present.
- [ ] Step 6 is present.
- [ ] Step 7 is present.
- [ ] Step 8 is present.
- [ ] Each step references prior conclusions where required.
- [ ] No step is skipped, merged, or reordered.

## Rule Integrity

- [ ] Enforcement tags are respected.
- [ ] Pattern, Strength, Climate, Disease, and Mediation priority is preserved.
- [ ] Pattern conflict with preliminary Useful God is declared when present.
- [ ] Climate override is used only under the protocol's stated condition.
- [ ] Mediation is used only as fallback under all required preconditions.
- [ ] Modern quantitative refinements do not override classical rules.

## Output Format

- [ ] Tendency and Condition is included.
- [ ] If-Then Switches are included.
- [ ] Execution Timing is included.
- [ ] Year-End Hard Check is included.
- [ ] The Hard Check contains observable evidence, pass condition, fail condition, and date.
- [ ] Outcomes use conditional language.
- [ ] No generic advice remains.
- [ ] No element-adding fix is present.
- [ ] Every conclusion is traceable to a stem-branch interaction or protocol step.
- [ ] Technical terms are translated or explained on first use.
- [ ] Final Tendency and Condition can be understood by a reader without classical Chinese metaphysics background.
- [ ] If-Then Switches use plain-language conditions, not only stem-branch notation.

## Rejection Triggers

- [ ] Fortune-telling language.
- [ ] Absolute prediction language.
- [ ] Missing Step 5 self-consistency check.
- [ ] Recalculated Luck Pillar table without user confirmation.
- [ ] Advice that could apply to anyone.
- [ ] Output without falsifiable evidence.
